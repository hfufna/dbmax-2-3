MaxScript CMD

Interactive MaxScript: the Command Prompt
MaxScript comes with this small app that will show the results of any statements you feed it.
Playing with MaxScript code in interactive sessions like this is a terrific way to learn the language.

How to use
You need to write some code in the command window in order to see it working. 
To get an idea on how the thing works just click on Quick start to load the default example, then click on Run.
If you don't know what to test and want to learn MaxScript at the same time just navigate the Sintax help, choose a language element (like USE) select the text of the example provided and copy (or drag) the selection to the black board, placing it ALWAYS between <% and%> tags.
Then click on Run.
If you get confused just click on Clear and start over again.

Important
The Command Prompt is not to be intended like a developement tool: you can't write application with it!
Applications are made of multiple pages linked togheter, while here you work always on the same page: anyway you can test here your code and save it to a text file for a later use.

This simple application is not an authoring tool for MaxScript: to make any serious work you still need to use a specific software as Microsoft FrontPage or Macromedia Dreamweaver.
The sole purpouse of this tool is to help you in your first steps with MaxScript, test the example included with MaxScript Programmer's Reference and eventually to do little experimenting of your own.

The black area of the screen is the command window: you can type your code there.
Code is not immediately available: you must save it before to try to run it.
This is done in the background by clicking the RUN button on top left of the screen.
When you click it the code you wrote is saved into a temporary file, then merged with header and footer supplied with the app (and that you can edit) to add the necessary MaxScript settings for execution and it is finally launched in a dedicated window (iframe).
Because of this mechanism, the application requires write permissions for its folder.

The environment
The code will run in a standard page making use the new dedicated MaxScript header 995: all webVars and libraries used by this header are available without the need to call it explicitly in the prompt window (as they are added at runtime by reading files header.txt and footer.txt located in the current app folder).
The default database for the command prompt is demo (i.e. issuing setDb("demo")) but obviously you can change it writing your own code.
 

Browser compatibility
This new version tries to address the previous issues caused by the pop-up blocking of recent browsers and those caused by the tabbed navigation by keeping all user interface in the same page using an iframe for the results. This way the results will be always shown in the same window. 
Anyway all browsers tend to cache files! The prompt page has the proper tags not to be cached, but some browser seems to ignore it.
In this case you will need to refresh manually before to see your code in action.

Important notice
This distribution does not necessarily include the Reference Archives (DBF files) needed by Sintax Help to work (or if included they may be outdated).
Be always shure to download the latest versions from the MaxScript web site.
